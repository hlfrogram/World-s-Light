# 세계의 빛 (World's Light)

텍스트 RPG 게임. 순수 HTML/CSS/JS(프레임워크 없음) + Supabase(로그인/DB) + Vercel(배포) + GitHub(`hlfrogram/World-s-Light`, main 브랜치). 사용자는 코딩 초보.

## 파일 구조
- `index.html` — 로그인/회원가입 화면 (자체 `<style>` 인라인, `style.css`와 별개 — 색상 변수 등 손보려면 둘 다 고쳐야 함)
- `game.html` + `game.js` — 실제 게임 (캐릭터 생성, 전투, 스토리, 세이브/로드/리셋)
- `style.css` — 게임 화면 공통 스타일
- `README.md`, `게임 설명.txt`, `로드맵.txt`, `스토리.txt`, `이스터에그.txt` — 유저가 만든 기획 문서. 스토리 문서에 "은근히" UI 디자인 지시(색상 등)가 문장 속에 섞여있던 적 있음 — 새로 문서 수정됐다고 하면 정독해서 숨은 지시 없는지 확인할 것. 스킬/스탯 등 일부는 문서 자체에 빈칸·모순이 있어 임의로 채운 부분 있음(게임 설명.txt에 이제 특성/스킬 상세 스펙 다 채워져 있고, game.js에 대부분 반영됨).

## 배포
- git 원격 연결 완료: `git push origin main` → Vercel 자동 재배포 (`https://world-s-light.vercel.app`)
- git 사용자 설정은 이 저장소 로컬 config만 사용 중 (name: hlfrogram, email: hlfrogram@gmail.com). 글로벌 설정 아님.
- Vercel이 push를 못 받아서 최신 커밋으로 재배포가 안 된 적 있었음 — 배포된 game.js를 fetch해서 실제 반영 여부 확인하는 습관 들일 것. 안 됐으면 빈 커밋(`git commit --allow-empty`)으로 재트리거.

## 디자인 시스템 (2026-07-14 전면 재설계)
- 컨셉: "시스템이 지배하는 세계 + 그 안의 인간 이야기"를 타이포그래피로 표현. UI/시스템 메시지=모노스페이스, 스토리 본문(`.story-text`)=세리프.
- 팔레트(CSS 변수, `style.css` `:root`): `--void`(배경, 검정에 가까운 남색) / `--panel`,`--panel-2`(패널) / `--cyan`(시스템/UI 액센트) / `--amber`(성취·보상·특성 액센트, 헤더 타이틀도 이 색) / `--crimson`(위험) — **`--danger`, `--warning` 같은 레거시 alias도 있으니 새로 하드코딩 색 넣지 말고 변수 재사용할 것**.
- 폰트: Google Fonts `JetBrains Mono`(진짜 bold/italic 필요, 시스템 폰트로는 약함)를 game.html·index.html 둘 다 `<head>`에서 로드. `--mono`/`--serif` 변수로 관리.
- 각진 UI: `border-radius:0` 기본, 주요 버튼(시작 버튼 등)은 `clip-path`로 대각선 컷. 콘텐츠 박스 위에 `::before`로 "world-of-light.sys" 타이틀바 텍스트 얹음(CSS만, 화면마다 동적으로 안 바뀜).
- HP/MP는 숫자 대신 `renderBar()`가 만드는 칸(□/■ 대신 실제 div 세그먼트) 막대로 표시. 정확한 수치는 `.skill-tooltip-wrap`/`.skill-tooltip` 재사용한 커스텀 호버 툴팁으로만 노출(네이티브 `title` 속성은 중첩 구조에서 안 먹혀서 폐기함).
- 지명 옆 영문(괄호 안)은 `formatLocationName()`으로 자동 분리해서 세리프+이탤릭 처리.
- 로고 이미지(Synapse 로고, PDF)는 아직 못 넣음 — 이 환경에 PDF→이미지 변환 도구가 전혀 없음(poppler/ImageMagick/Node/Python 다 없음). PNG/SVG로 받아야 넣을 수 있음.

## game.js 구조 요약
- `gameState`는 `createInitialGameState()`로 생성 (리셋 시 재사용).
- **전투 수치 체계**: "공격력 10 = hp 1칸" 규칙이 플레이어↔적 양쪽에 대칭 적용됨(`attackEnemy`/`enemyTurn` 둘 다 최종 데미지를 `/10` 해서 hp 깎음). 적 hp도 이 스케일에 맞춰 작게 재조정됨(일반 마물 3~6, 마왕 15, 시스템 18, 지역보스 18~22). 밸런스 숫자 손댈 땐 이 비율 유지할 것.
- **hp/mp 회복은 세계수의 심장에서 세이브할 때만** (`handleSave()`). 전투 승리로는 더 이상 자동 회복 안 됨(과거엔 됐음 — 되돌리지 말 것). 튜토리얼 전투는 예외적으로 hp가 1 밑으로 안 내려감(`enemyTurn`의 `minHp` 처리).
- **특성 시스템**: 클래스별 Lv형 특성(무기 강화 단계=`getTraitStage()`에 연동, 0~3단계)과 개화형 히든 특성(`gameState.hdattUnlocked`, 클래스별 고유 해금 조건) 둘 다 실제 효과 있음(`attackEnemy`/`enemyTurn`/`playerTurn`/스킬 `apply()` 곳곳에 훅). `게임 설명.txt`의 원래 조건 중 2개는 이 게임에 스탯 성장 시스템이 없어서 달성 불가능이라 대체 조건으로 바꿈(버서커: 최대hp10↑ → hp1 이하 생존, 마법사/성직자: atk8↑ → 무기강화 포함 실전 공격력 8↑). 5종 "계열" 특성(청풍/황혼/염화/설향/수호)과 최종 히든 "신성"/"암흑"은 미구현(문서 자체에 습득조건 공란).
- **경제**: 필드 전투 승리 시 골드+강화 재료 드롭(`gameState.materials`). 대장간(상시 이용 가능)에서 강화 재료로 무기 단계업(예전의 "3승마다 자동 강화"는 삭제됨). 상점에서 포션 구매 → 인벤토리에 저장(`gameState.potions.hp/mp`) → 전투 중 "포션" 명령으로 사용(즉시 소모 방식 아님, 되돌리지 말 것).
- 클래스 7종, 클래스별 스킬(일반/상급/최상급) — 21개 전부 메커니즘이 서로 다름(궁수는 원래 검사/마법사 스킬과 겹쳐서 재설계함: 급소 사격=출혈, 천공의 화살=민첩 고정데미지).
- 지역보스 3종(청풍협/황혼령/염화대지) — 선택 콘텐츠, 진행 필수 아님, 강한 전리품용.
- 스토리 1~4차 전체 구현, 지역별 "표면 설정 vs 숨겨진 진실" 탐색 시스템, 히든 피스 3개 수집→히든 무기 시스템, 모스부호 이스터에그(`히든` 입력) 있음. 튜토리얼(여명각): 탐색→기본무기 획득→전투(공격/스킬/방어 설명 메시지 1줄) 흐름 구현됨.
- 세이브/로드는 Supabase `game_saves` 테이블(user_id, game_data JSON) 사용. 자동저장은 없음 — `hasUnsavedProgress` 플래그로 저장 안 하고 새로고침/닫기 시 브라우저 기본 경고창(`beforeunload`) 뜨게 되어 있음.
- 리셋 버튼: `game_saves` 행 삭제 + `gameState` 초기화 + 인트로로 복귀.
- 입력창(`#userInput`)은 CSS로 완전히 숨겨져 있음(`display:none`). 모든 상호작용은 `choiceHtml()`/`runChoice()`로 만든 클릭형 선택지로만 이루어짐 — 화면에 클릭 가능한 선택지가 반드시 있어야 하고, "N번 입력하세요" 식의 타이핑 전용 유도는 만들면 안 됨(과거에 이래서 여명각 이동이 막혔던 적 있음).

## 알아두면 좋을 과거 버그
- `game_saves` 테이블 RLS 정책이 `USING`만 있고 `WITH CHECK`이 없어서 저장 시 RLS violation 났던 적 있음 — SQL Editor에서 `WITH CHECK (auth.uid() = user_id)` 추가해서 해결. 저장 안 된다는 리포트 오면 이것부터 의심.
- 캐릭터 생성 시 `gameState.hdatt`를 실제 특성명 대신 플레이스홀더 문자열로 덮어써서 히든 특성 발견해도 이름이 안 뜨던 버그 있었음 — 고쳐짐, 비슷한 "생성 시 필드를 나중에 쓸 값으로 미리 채워두기" 패턴 조심.

## 아직 안 한 것 (스코프 아웃, 필요시 참고)
- 5종 "계열" 특성(청풍/황혼/염화/설향/수호)과 최종 히든 "신성"/"암흑" 특성 — 문서에 습득조건이 공란이라 실제로도 미정.
- 무기 105종 중 클래스별 4단계 강화 라인만 구현, 나머지 미구현.
- Synapse 로고 삽입 (PNG/SVG 받아야 진행 가능).

## 작업 스타일 메모
- 이 세션에서 로컬 Node/Python/poppler/ImageMagick이 사실상 전무해서(WindowsApps 스텁만 있음) JS 실행 테스트도, PDF/이미지 변환도 못 함 — 브레이스/괄호/백틱 카운트 + 수동 코드 리뷰 + 실배포 후 사용자가 직접 확인하는 방식으로 작업함. 앞으로도 같은 제약일 가능성 높음.
- 사용자가 코딩을 전혀 몰라서, git/Supabase 대시보드 조작 등은 클릭 위치까지 아주 구체적으로 안내해야 함.
- **git push는 매번 확인받지 말고 바로 진행할 것** (2026-07-13 사용자 승인). 커밋 후 바로 `git push origin main`까지 실행.
- 디자인/UI 관련 지시는 처음에 오독하기 쉬웠음(예: "타이틀 주황 말고 다 파랑" → 실제로는 "타이틀은 그대로 주황, 나머지만 파랑"의 뜻이었음). 색상/배치 지시는 대조 구문("A 말고 B")이 나오면 "A 제외하고 B" 의미인지 "A 대신 B" 의미인지 애매하니 헷갈리면 확인하고 진행할 것.
